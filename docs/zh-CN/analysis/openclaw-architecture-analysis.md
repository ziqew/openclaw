# OpenClaw 架构分析与使用场景

## 一、项目概述

**OpenClaw** 是一个开源的个人 AI 助手平台，基于 TypeScript 构建，可在本地设备上运行。它通过统一的网关架构，将 AI 能力集成到 20+ 个消息平台中，实现跨平台的智能对话体验。

### 核心特性

- **多平台支持**：支持 WhatsApp、Telegram、Discord、Slack、Signal、iMessage 等 20+ 个消息平台
- **本地优先**：数据和会话完全在本地设备上处理，保护隐私
- **插件化架构**：通过扩展系统轻松添加新的消息平台和功能
- **多模型支持**：集成 Anthropic Claude、OpenAI、Google AI、AWS Bedrock 等多个 AI 提供商
- **跨设备访问**：支持 macOS、iOS、Android、Web、CLI 等多种客户端

### 技术栈

- **运行时**：Node.js ≥22（主要）、Bun（开发/测试）
- **编程语言**：TypeScript（ESM，严格类型检查）
- **构建工具**：tsdown、Rolldown
- **测试框架**：Vitest（V8 覆盖率）
- **包管理器**：pnpm（主要）

### 2.2 核心模块详解

#### **网关 (Gateway)** - `src/gateway/`

网关是 OpenClaw 的核心控制平面，负责：

- **连接管理**：WebSocket/HTTP 服务器，处理客户端连接
- **会话管理**：维护对话状态和上下文
- **代理编排**：调度和管理 AI 代理
- **配置热重载**：动态更新配置无需重启
- **定时任务**：Cron 作业调度
- **Hook 执行**：事件驱动的自动化

**关键文件**：
- `server.impl.ts` - 网关服务器主实现
- `server-methods/` - RPC 方法处理器
- `protocol/` - 网关协议定义

#### **AI 代理 (Agents)** - `src/agents/`

AI 代理运行时系统提供：

- **Pi 代理运行器**：集成 Pi AI 代理框架
- **工具系统**：Bash、Read、Write、Edit、Grep 等工具
- **认证管理**：OAuth/API 密钥管理，支持故障转移
- **模型选择**：多提供商支持（Anthropic、OpenAI、Google、Bedrock）
- **使用跟踪**：模型使用情况统计

**关键能力**：
- 多提供商认证配置文件轮换
- 模型故障转移链
- CLI 命令执行
- 上下文管理

#### **消息平台 (Channels)** - `src/channels/`、`extensions/`

支持 20+ 个消息平台的插件化架构：

**内置平台**：
- WhatsApp（Baileys web provider）
- Telegram（grammY bot）
- Discord（Discord.js）
- Slack（Bolt SDK）
- Signal（signal-cli 封装）
- iMessage（macOS，传统）

**扩展平台**（插件）：
- BlueBubbles、Matrix、MS Teams、Google Chat
- Feishu、LINE、Mattermost、Nextcloud Talk
- Nostr、Tlon、Twitch、Zalo
- 语音通话等

**平台特性**：
- 统一的插件适配器接口
- 白名单/黑名单管理
- 私聊/群聊策略
- 媒体支持（因平台而异）
- 输入指示器
- 反应确认

#### **配置系统 (Config)** - `src/config/`

统一的配置管理系统：

- **格式**：JSON5，支持环境变量替换
- **验证**：基于 Zod 的模式验证
- **结构**：
  ```json
  {
    "agents": { "defaults", "list", "bindings" },
    "gateway": { "mode", "host", "port", "auth" },
    "whatsapp": {...},
    "telegram": {...},
    "models": { "providers" },
    "plugins": { "enabled", "config" },
    "hooks": [],
    "cron": { "jobs" }
  }
  ```
- **存储位置**：
  - 配置：`~/.openclaw/config.json`
  - 会话：`~/.openclaw/sessions/`
  - 凭证：`~/.openclaw/credentials/`

#### **自动回复 (Auto-Reply)** - `src/auto-reply/`

消息处理和响应生成：

- 回复生成逻辑（带历史管理）
- 大消息分块
- 响应模板化
- 渐进式输入指示
- 媒体附件处理

#### **媒体处理 (Media)** - `src/media/`

媒体文件处理管道：

- MIME 类型检测
- 媒体存储管理
- 视觉模型集成
- 媒体分析提供商适配器

#### **基础设施 (Infra)** - `src/infra/`

跨领域关注点：

- **网络**：SSRF 保护、fetch 防护
- **TLS**：证书管理
- **出站**：消息发送基础设施
- **诊断**：遥测事件
- **更新**：版本检查和启动更新
- **环境**：环境变量管理

## 三、核心功能

### 3.1 AI 对话能力

- **多模型支持**：Anthropic Claude、OpenAI GPT、Google Gemini、AWS Bedrock 等
- **上下文管理**：自动维护对话历史和上下文
- **流式响应**：实时流式输出，提升用户体验
- **工具调用**：代理可执行 Bash 命令、读写文件、搜索代码等

### 3.2 消息平台集成

- **20+ 平台支持**：覆盖主流即时通讯应用
- **统一接口**：通过插件系统抽象平台差异
- **媒体处理**：支持图片、视频、文档等多媒体内容
- **群聊管理**：支持群聊场景，可配置提及触发

### 3.3 本地优先架构

- **隐私保护**：所有数据在本地处理，不上传云端
- **离线能力**：网关可完全离线运行（需要 AI 模型访问权限）
- **自托管**：完全掌控数据和服务

### 3.4 自动化与编排

- **Hooks**：事件驱动的自动化（Gmail 监听、Webhooks 等）
- **Cron 任务**：定时任务调度，支持独立代理
- **命令路由**：智能消息路由到不同代理

### 3.5 跨平台客户端

- **macOS 应用**：Swift/SwiftUI 菜单栏应用，集成网关守护进程
- **iOS 应用**：SwiftUI 移动应用
- **Android 应用**：Kotlin 应用
- **Web UI**：基于 Vite + Lit 的 Web 界面
- **CLI**：功能完整的命令行工具

### 3.6 扩展性

- **插件系统**：通过插件 SDK 轻松扩展功能
- **频道插件**：添加新的消息平台集成
- **认证插件**：添加新的 OAuth 提供商
- **功能插件**：内存后端、诊断、LLM 任务编排等

## 七、部署方式

### 7.1 本地开发

```bash
# 安装依赖
pnpm install

# 构建项目
pnpm build

# 运行 CLI
pnpm openclaw

# 启动网关
pnpm openclaw gateway run
```

### 7.2 生产部署

```bash
# 全局安装
npm install -g openclaw

# 初始化配置
openclaw onboard

# 启动网关（作为守护进程）
openclaw gateway run --daemon
```

### 7.3 Docker 部署

支持 Docker 容器化部署，适用于服务器环境。

### 7.4 树莓派部署

专门优化的树莓派部署方案，可作为家庭 AI 中心。

无论是个人使用、团队协作、开发辅助还是智能家居，OpenClaw 都能提供强大而灵活的解决方案。其开源特性和活跃的社区支持，使其成为构建个人 AI 基础设施的理想选择。
